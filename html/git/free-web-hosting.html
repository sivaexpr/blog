Create a Page in Github

Creating a page in Githib is a first step to you to host your files. Follow Steps mentioned in page. Don’t forget to create your repo with .github.io suffix. For example, if you want to create a repository something like techjira then use name as techjira.github.io. if you don’t give this suffix, it will still work but the default page will be rendered from README.md file. If you give a suffix, it will pick index.html as a default page.

configuration Github with custom domain

To map your custom domain, you should know how to add DNS records. If you don’t know how to do it, contact your domain provider. But most of the domain providers will have a web page to manage your DNS records. Here I am using godaddy DNS page to give the explanation. There is a good documentation avalable how to use custom URL with Github pages but you need to go through multiple links to achieve this. There are two different type of domains i.e., sub domain and apex domain. Sub domain usually comes with www. For example www.techjira.com is sub domain. Apex domain doesn’t have www. For example http://techjira.com is a apex domain. DNS configuration is different for sub domains and apex. Before doing configuration first add your domain in github page.

  1. Login into Gitgub and select the page from the repositories and click on Settings
  2. Scroll down until you see Github pages. If you already not yet enabled the pages, select the branch to keep your web pages. Most of the time its Master and click on Save button.
  3. Once you enabled the page option, give the domain name and click on Save button. You will see the success message with your website name. The moment you save domain, you will have a file called CNAME in your repository.

Once you are done with Github configuration go to DNS configuration.

Configure domain with DNS provider

  1. First login into DNS provider website and go to DNS setting
  2. Remove all the entries if you have any. Make sure you don’t have A records or CNAME records
  3. If you are using sub domain(WWW) add an entry of CNAME with value as YOUR-GITHUB-USERNAME.github.io. for example, if you have a Github user as “test” and then the entry should be test.gitgub.io
  4. If you have apex domain, then add add entry of A with the IP address of 192.30.252.153 or 192.30.252.154.
  5. If you have both sub domain and apex then you need both entries
  6. Once your configuration is done access your website for browser. Please note that some DNS providers will take upto 1 day to refresh your records. If doesn’t work wait for some time and try again. Below are the steps to verify your configuration

Verify configuration

You can verify the configuration you did using multiple ways. You can also check online. Go to https://toolbox.googleapps.com/apps/dig/ and click on A OR CNAME to verify the records.
